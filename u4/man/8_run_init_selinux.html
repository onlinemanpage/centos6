NAME run_init_selinux - Security Enhanced Linux Policy for the run_init pro- cesses

DESCRIPTION Security-Enhanced Linux secures the run_init processes via flexible mandatory access control.

The run_init processes execute with the run_init_t SELinux type. You can check if you have these processes running by executing the ps com- mand with the -Z qualifier.

For example:

ps -eZ | grep run_init_t

ENTRYPOINTS The run_init_t SELinux type can be entered via the "run_init_exec_t" file type. The default entrypoint paths for the run_init_t domain are the following:"

/usr/sbin/run_init

PROCESS TYPES SELinux defines process types (domains) for each process running on the system

You can see the context of a process using the -Z option to ps

Policy governs the access confined processes have to files. SELinux run_init policy is very flexible allowing users to setup their run_init processes in as secure a method as possible.

The following process types are defined for run_init:

run_init_t

Note: semanage permissive -a run_init_t

can be used to make the process type run_init_t permissive. Permissive process types are not denied access by SELinux. AVC messages will still be generated.
