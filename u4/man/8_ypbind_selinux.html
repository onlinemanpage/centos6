NAME ypbind_selinux - Security Enhanced Linux Policy for the ypbind pro- cesses

DESCRIPTION Security-Enhanced Linux secures the ypbind processes via flexible mandatory access control.

The ypbind processes execute with the ypbind_t SELinux type. You can check if you have these processes running by executing the ps command with the -Z qualifier.

For example:

ps -eZ | grep ypbind_t

ENTRYPOINTS The ypbind_t SELinux type can be entered via the "ypbind_exec_t" file type. The default entrypoint paths for the ypbind_t domain are the following:"

/sbin/ypbind, /usr/sbin/ypbind

Note: semanage permissive -a ypbind_t

can be used to make the process type ypbind_t permissive. Permissive process types are not denied access by SELinux. AVC messages will still be generated.

BOOLEANS SELinux policy is customizable based on least access required. ypbind policy is extremely flexible and has several booleans that allow you to manipulate the policy and run ypbind with the tightest access possible.

If you want to allow system to run with NIS, you must turn on the allow_ypbind boolean.

setsebool -P allow_ypbind 1

If you want to allow system to run with NIS, you must turn on the allow_ypbind boolean.

setsebool -P allow_ypbind 1
