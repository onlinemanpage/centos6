NAME kudzu_selinux - Security Enhanced Linux Policy for the kudzu processes

DESCRIPTION Security-Enhanced Linux secures the kudzu processes via flexible manda- tory access control.

The kudzu processes execute with the kudzu_t SELinux type. You can check if you have these processes running by executing the ps command with the -Z qualifier.

For example:

ps -eZ | grep kudzu_t

ENTRYPOINTS The kudzu_t SELinux type can be entered via the "mtrr_device_t,unla- beled_t,proc_type,sysctl_type,filesystem_type,file_type,kudzu_exec_t" file types. The default entrypoint paths for the kudzu_t domain are the following:"

/dev/cpu/mtrr, all files on the system, /sbin/kudzu, /sbin/kmodule, /usr/sbin/kudzu

PROCESS TYPES SELinux defines process types (domains) for each process running on the system

You can see the context of a process using the -Z option to ps

Policy governs the access confined processes have to files. SELinux kudzu policy is very flexible allowing users to setup their kudzu pro- cesses in as secure a method as possible.

The following process types are defined for kudzu:

kudzu_t

Note: semanage permissive -a kudzu_t

can be used to make the process type kudzu_t permissive. Permissive process types are not denied access by SELinux. AVC messages will still be generated.

Policy governs the access confined processes have to these files. SELinux kudzu policy is very flexible allowing users to setup their kudzu processes in as secure a method as possible.

The following file types are defined for kudzu:

kudzu_exec_t

- Set files with the kudzu_exec_t type, if you want to transition an executable to the kudzu_t domain.

kudzu_tmp_t

- Set files with the kudzu_tmp_t type, if you want to store kudzu tem- porary files in the /tmp directories.

kudzu_var_run_t

- Set files with the kudzu_var_run_t type, if you want to store the kudzu files under the /run directory.

Note: File context can be temporarily modified with the chcon command. If you want to permanently change the file context you need to use the semanage fcontext command. This will modify the SELinux labeling database. You will need to use restorecon to apply the labels.
