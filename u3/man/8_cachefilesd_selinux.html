NAME cachefilesd_selinux - Security Enhanced Linux Policy for the cachefilesd processes

DESCRIPTION SELinux Linux secures cachefilesd (policy for cachefilesd) processes via flexible mandatory access control.

FILE CONTEXTS SELinux requires files to have an extended attribute to define the file type.

You can see the context of a file using the -Z option to ls

Policy governs the access confined processes have to these files. SELinux cachefilesd policy is very flexible allowing users to setup their cachefilesd processes in as secure a method as possible.

The following file types are defined for cachefilesd:

cachefiles_var_t

- Set files with the cachefiles_var_t type, if you want to store the cachef files under the /var directory.

Paths: /var/run/cachefilesd.pid, /var/fscache(/.*)?, /var/cache/fscache(/.*)?

cachefilesd_exec_t

- Set files with the cachefilesd_exec_t type, if you want to transition an executable to the cachefilesd_t domain.

cachefilesd_var_run_t

- Set files with the cachefilesd_var_run_t type, if you want to store the cachefilesd files under the /run directory.

Note: File context can be temporarily modified with the chcon command. If you want to permanently change the file context you need to use the semanage fcontext command. This will modify the SELinux labeling database. You will need to use restorecon to apply the labels.
