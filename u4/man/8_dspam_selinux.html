NAME dspam_selinux - Security Enhanced Linux Policy for the dspam processes

DESCRIPTION Security-Enhanced Linux secures the dspam processes via flexible manda- tory access control.

The dspam processes execute with the dspam_t SELinux type. You can check if you have these processes running by executing the ps command with the -Z qualifier.

For example:

ps -eZ | grep dspam_t

ENTRYPOINTS The dspam_t SELinux type can be entered via the "dspam_exec_t" file type. The default entrypoint paths for the dspam_t domain are the fol- lowing:"

/usr/bin/dspam

PROCESS TYPES SELinux defines process types (domains) for each process running on the system

You can see the context of a process using the -Z option to ps

Policy governs the access confined processes have to files. SELinux dspam policy is very flexible allowing users to setup their dspam pro- cesses in as secure a method as possible.

The following process types are defined for dspam:

dspam_t

Note: semanage permissive -a dspam_t

can be used to make the process type dspam_t permissive. Permissive process types are not denied access by SELinux. AVC messages will still be generated.

Policy governs the access confined processes have to these files. SELinux dspam policy is very flexible allowing users to setup their dspam processes in as secure a method as possible.

The following file types are defined for dspam:

dspam_exec_t

- Set files with the dspam_exec_t type, if you want to transition an executable to the dspam_t domain.

dspam_initrc_exec_t

- Set files with the dspam_initrc_exec_t type, if you want to transi- tion an executable to the dspam_initrc_t domain.

dspam_log_t

- Set files with the dspam_log_t type, if you want to treat the data as dspam log data, usually stored under the /var/log directory.

dspam_tmp_t

- Set files with the dspam_tmp_t type, if you want to store dspam tem- porary files in the /tmp directories.

dspam_var_lib_t

- Set files with the dspam_var_lib_t type, if you want to store the dspam files under the /var/lib directory.

dspam_var_run_t

- Set files with the dspam_var_run_t type, if you want to store the dspam files under the /run directory.

Note: File context can be temporarily modified with the chcon command. If you want to permanently change the file context you need to use the semanage fcontext command. This will modify the SELinux labeling database. You will need to use restorecon to apply the labels.
