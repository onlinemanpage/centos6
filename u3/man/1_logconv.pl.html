NAME logconv.pl - analyzes Directory Server access log files

SYNOPSIS logconv.pl [-h] [-d <rootDN>] [-s <size limit>] [-v] [-V] [-S <start time>] [-E <end time>] [-efcibaltnxgju] [ access log ... ... ]

DESCRIPTION Analyzes Directory Server access log files for specific information defined on the command line

OPTIONS A summary of options is included below:

-h help/usage

-d <Directory Managers DN> DEFAULT -> cn=directory manager

-s <Number of results to return per category> DEFAULT -> 20

-X <IP address to exclude from connection stats> E.g. Load balancers

-v show version of tool Print version of the tool

-S <time to begin analyzing logfile from> Time to begin analyzing logfile from E.g. [28/Mar/2002:13:14:22 -0800]

-E <time to stop analyzing logfile> Time to stop analyzing logfile from E.g. [28/Mar/2002:13:24:62 -0800]

-V <enable verbose output - includes all stats listed below> Verbose output

-[efcibaltnxgju] e Error Code stats f Failed Login Stats c Connection Code Stats i Client Stats b Bind Stats a Search Base Stats l Search Filter Stats t Etime Stats n Nentries Stats x Extended Operations r Most Requested Attribute Stats g Abandoned Operation Stats j Recommendations u Unindexed Search Stats y Connection Latency Stats p Open Connection ID Stats

USAGE Examples:

logconv.pl -s 10 -V access

logconv.pl -d "cn=directory manager" /export/server4/slapd-host/logs/access*

logconv.pl -s 50 -ibgju access*

logconv.pl -S "[28/Mar/2002:13:14:22 -0800]" -E "[28/Mar/2002:13:50:05 -0800]" -e access
